Non-linear movement

Problem

During the development process, I struggled with how to make the image inside the moving sphere change only for a short moment when it touches the edge of the canvas. I wanted the pattern inside the sphere to react to the collision visually, instead of changing permanently or requiring user interaction. At first, I did not know how to trigger a temporary image change that would automatically return to the original image while keeping the animation smooth.

Solution

To solve this problem, I used a simple frame based countdown system. When the sphere hits the top edge of the canvas, I switch the displayed image by changing the currentimage variable and set a short countdown value. Inside the draw loop, the countdown decreases every frame. Once the countdown reaches zero, the image switches back to the original one. This approach allows the image change to feel responsive and temporary without interrupting the animation.

        if (y - r < 0) {
          y = r - (y - r);
          currentimage = imagelebubu;
          countDown = 16;
        }
        
        if (countDown > 0) {
          countDown--;
        } else {
          currentimage = imglebu;
